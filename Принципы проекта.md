# Отчёт по принципам проекта PyType

## 1. Архитектура — строгий MVC + Service Layer

Корень разбит на 4 пакета, **каждый со своей ответственностью**:

```
project/
├── main.py                  # ТОЛЬКО точка входа: QApplication + загрузка QSS + старт контроллера
├── controllers/             # связь View ↔ Service, обработка событий, навигация
├── models/                  # dataclass'ы данных + провайдеры (источники)
├── services/                # чистая бизнес-логика, без Qt
├── views/                   # ТОЛЬКО виджеты PyQt6, один файл = один экран
└── resources/
    ├── styles.qss           # ВСЁ оформление
    └── data/                # JSON-файлы (не БД)
```

**Правило:** один файл — одна ответственность. Никаких "God-классов", где смешаны UI и логика.

---

## 2. Жёсткое разделение слоёв (Strict Separation)

- **services/** не импортирует ничего из PyQt6 — это чистый Python. Это даёт переносимость и тестируемость.
- **views/** не содержит бизнес-логики (никаких расчётов, чтения файлов).
- **controllers/** — посредник: View испускает сигнал → Controller вызывает Service → результат обратно во View.
- **models/** — только структуры данных (`@dataclass`) и поставщики (`WordProvider`).

Пример из PyType: `EngineService` считает WPM/Accuracy без единого `import PyQt6`. `TypingView` только показывает.

---

## 3. Стилизация — только через QSS

Это **главное правило** проекта (см. `README.md`, раздел "Правилы"):

- Все цвета, шрифты, отступы, скругления — **только** в [resources/styles.qss](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/resources/styles.qss).
- В Python-коде **запрещён** `setStyleSheet()`, кроме редких случаев динамики (например, мигание ошибки).
- Загрузка QSS — централизованно в [main.py:13-17](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/main.py#L13-L17) через `QApplication.setStyleSheet()`.
- В начале QSS-файла — **палитра в комментарии** (см. [styles.qss:1-3](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/resources/styles.qss#L1-L3)). Все цвета берутся из неё, никаких "магических" hex-кодов.

---

## 4. Naming convention — `objectName` для всех виджетов

Каждый виджет, который должен быть стилизован, получает `setObjectName("snake_case")`:

```python
self.btn_nav_typing.setObjectName("nav_btn")
self.lbl_wpm.setObjectName("stat_value_accent")
```

В QSS селектор работает по этому имени:

```qss
QPushButton#nav_btn { ... }
QLabel#stat_value_accent { ... }
```

**Зачем:** позволяет иметь несколько `QPushButton` с разным видом, не перетягивая стили глобально.

---

## 5. Layout-only, никаких координат

- **Запрещено** использовать `widget.move(x, y)` и `setGeometry()`.
- Всё располагается через `QVBoxLayout` / `QHBoxLayout` / `QGridLayout`.
- Для центрирования — `addStretch()` слева и справа (см. [typing_view.py:147-149](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/views/typing_view.py#L147-L149)).
- Для отступов — `setContentsMargins()` и `setSpacing()`, **не** через пустые QLabel.

---

## 6. Структура View — приватные `_build_*` методы

`TypingView` — эталон: `init_ui()` собирает корневой layout, а каждая секция строится отдельным методом:

```python
def init_ui(self):
    root.addWidget(self._build_progress_bar())
    root.addWidget(self._build_header())
    root.addWidget(self._build_main(), stretch=1)
    root.addWidget(self._build_footer())

def _build_header(self): ...
def _build_logo(self): ...
def _build_nav(self): ...
```

**Зачем:** каждая часть UI читается изолированно, легко переставлять блоки, не растёт мега-функция.

---

## 7. Данные — только через JSON, без БД

- Все ресурсы лежат в [resources/data/](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/resources/data/) (`words.json`, `history.json`).
- Чтение/запись — через отдельный `services/storage_service.py` (никто другой к файлам не лезет).
- Пути строятся через `pathlib.Path(__file__).parent.parent / ...` — **относительно файла**, а не CWD (см. [word_provider.py:7](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/models/word_provider.py#L7)).

---

## 8. Модели данных — `@dataclass`

Никаких "сырых dict'ов" между слоями. Пример [test_result.py](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/models/test_result.py):

```python
@dataclass
class TestResult:
    date: str
    wpm: int
    accuracy: float
    error_count: int
    duration: int
    mode: str
```

В новом проекте по аналогии: `Note`, `CalculationHistoryEntry`, `Track` и т.д.

---

## 9. Навигация через QStackedWidget

Один `QMainWindow` + `QStackedWidget` как central widget. Все экраны — `QWidget`-подклассы, добавляются через `addWidget(...)`. Переключение — `setCurrentIndex(...)` из `NavigationController`. Никаких отдельных окон (`QDialog`) для основных экранов.

См. [main_window.py](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/views/main_window.py) и [navigation_controller.py](vscode-webview://0pd5ubmj1laka2n6ch4jgqhoov05ha5qsdf6mp4okttkhoopt6ol/controllers/navigation_controller.py).

## 12. main.py — минимальный

```python
def main():
    app = QApplication(sys.argv)
    load_styles()
    nav_controller = NavigationController()
    sys.exit(app.exec())
```

Никакой логики в `main.py` — только запуск и подгрузка стилей. Всё остальное делает `NavigationController`.

## Что передать новому ИИ (TL;DR-инструкция)

> Проект на Python + PyQt6 строится по схеме MVC + Services. Соблюдай эти принципы:
> 
> 1. **MVC + Services** — `controllers/`, `models/`, `services/`, `views/`, `resources/`. Один файл — одна ответственность.
> 2. **services/ без PyQt6** — чистая логика отдельно от UI.
> 3. **Стилизация только в `resources/styles.qss`** — никаких `setStyleSheet()` в Python (кроме динамики).
> 4. **Палитра в комментарии** в начале QSS, все цвета — оттуда.
> 5. **`setObjectName("snake_case")`** на каждом виджете, который надо стилизовать; селектор в QSS — `QType#object_name`.
> 6. **Только Layout** — `QVBoxLayout`/`QHBoxLayout`/`QGridLayout`. Запрещены `move()`, `setGeometry()`.
> 7. **View — приватные `_build_*` методы**; `init_ui()` только собирает.
> 8. **Один экран = один файл** в `views/`. Никаких "God Widgets".
> 9. **Навигация** — `QMainWindow` + `QStackedWidget` + `NavigationController`.
> 10. **Данные** — JSON в `resources/data/`, доступ только через `services/storage_service.py`.
> 11. **Модели** — `@dataclass`, не голые dict'ы.
> 12. **Type hints** на всех публичных методах.
> 13. **Пути к файлам** — `Path(__file__).parent / ...`, не относительно CWD.
> 14. **`main.py`** — только точка входа: `QApplication`, загрузка стилей, старт контроллера.
> 15. Комментарии — только "почему", не "что".
